Concepts

A small vocabulary for a common kernel shape.

Trellis is designed around explicit dependency identity, structural collection diffs, scoped resource ownership, and revisioned materialized output.

Input node

o Canonical host-owned fact changed only through transactions.

Derived node

<> Deterministic value from declared dependencies.

Collection node

[#] Derived set or map that produces structural diffs.

Diff

+/- Added, removed, updated, and unchanged members in deterministic order.

Resource plan

-> Plain commands that the host actor applies after propagation.

Scope

[[ ]] Lifetime owner for resources, outputs, and dependency attachments.

Revision

#42 Coherent output version emitted with frames and audit data.

Materialized output

External-facing baseline, delta, clear, and rebaseline frames.

Full recompute oracle

A test hook that checks incremental state against a recomputed truth model.
