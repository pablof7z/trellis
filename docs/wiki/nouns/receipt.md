# receipt

An NMP-owned, Trellis-vocabulary-free data shape that carries the diagnostic information Chirp/agents consume (projection key, view/scope label, interest shape, owner key, owner count transition, relay rows, structured reason codes, transaction id/revision/sequence, timestamps, command kind, outcome status, parent scope) — Chirp renders receipts but never parses Trellis keys or receives ResourcePlan/ResourceKey/Graph.
