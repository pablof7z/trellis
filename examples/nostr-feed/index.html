Example 5

Nostr feed demand

Nostr is a strong example of the pattern, not the brand center. Account state derives follow sets, feed demand, subscriptions, and feed output.

active_account
 -> follow_set
 -> feed_demand
 -> relay_subscriptions
 -> feed_output #31

Problem

Account switches and contact-list replacement must withdraw old source demand and rebaseline projections.

Trace

Follow set shrinks from {A, B, C} to {B, C, D}. Close A filters, open D filters, preserve B/C where possible, emit a projection revision.

Invariant

Source shrink must withdraw resources. Stale feed rows must not survive as visible output.

Boundary

The host owns relay I/O and retries. Trellis computes demand and output frames.
