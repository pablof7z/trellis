Trellis

Your app already has a hidden resource graph.

Signals recompute values. Trellis reconciles resources.

An early Rust runtime for application kernels where changing state must safely open, close, cancel, clear, replay, and rebaseline live resources.

Read the guide See the traces GitHub
No hidden I/O in graph propagation. No orphan resources after scope close. No wildcard demand from empty sets.

The bug class

The bug is not that a value failed to recompute. The bug is that the old value still owns something.

A workspace changes, but old sync windows remain open.

A file is deleted, but diagnostics remain visible.

A plugin is disabled, but its worker keeps running.

A filter becomes empty, but a wildcard subscription opens.

A panel closes, but its live query stays alive.

These are resource graph bugs. Trellis turns them into explicit diffs, plans, scopes, and output revisions.

The model

Canonical input derives collections. Collection changes produce structural diffs. Diffs produce resource plans. Plans update materialized output revisions.

input->derived collection->structural diff->resource plan->output revision

Graph phase

Stage inputs, validate dependencies, derive dirty nodes, diff collections, plan commands, and commit revisions.

  • no external I/O
  • no task spawning
  • no host callbacks
  • no hidden handle mutation

Host phase

Apply returned commands, emit returned frames, observe resource results, and feed those results back as inputs.

  • commands are data
  • frames are inspectable
  • runtime results are explicit
  • logs explain each revision

Concrete use cases

The same primitive set handles network subscriptions, file watchers, plugin resources, and output surfaces.

Local-first sync

workspace -> projects -> sync windows -> issue board

Switching workspaces closes A, keeps B, opens C, replays cached rows, and emits a coherent board revision.

MQTT dashboard

site filter -> devices -> topics -> dashboard cards

Filter shrink unsubscribes removed devices. Empty sets subscribe to nothing.

Plugin runtime

enabled plugins -> commands/watchers/workers -> shell output

Disable means teardown: no stale commands, no orphan workers, no forgotten file watchers.

Language server

files -> module graph -> watchers -> diagnostics

Salsa can recompute pure diagnostics. Trellis models clear diagnostics, update watcher sets, and cancel analysis jobs.

Nostr/NMP feed

account -> contacts -> relay filters -> feed projection

Contact-list replacement withdraws old demand and rebaselines visible output.

Market data

watchlist -> quote streams -> alert jobs -> terminal output

A later pressure test for performance, entitlement, and backpressure boundaries.

Correctness is the product.

Source shrink withdraws

Removed collection members produce close commands.

Empty fails closed

Empty demand stays empty unless a fallback is explicit.

Scope close releases

A resource with no owning scope is a bug.

Revision is coherent

Consumers receive baseline, delta, clear, or rebaseline frames.

Plan is inspectable

Your tests and logs can read the same command data the host applies.

Incremental equals truth

Full-recompute oracles check incremental graph state.

When not to use Trellis

Do not use Trellis for ordinary UI state, pure memoization, or stream transformations that do not own external resources. Use it when a derived value opens something, closes something, or exposes revisioned output that must stay coherent.

let tx = graph.transaction(|tx| {
    tx.set(selected_site, SiteId("plant-12"));
});

for command in tx.commands() {
    runtime.apply(command);
}

for frame in tx.frames() {
    output.send(frame);
}

Trellis is for Rust developers who have hand-rolled callback glue to keep derived state, subscriptions, jobs, caches, and output surfaces in sync.
