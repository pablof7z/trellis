A deterministic reconciler for Rust.

State changes go in. Resource commands, output frames, and an auditable receipt come out. Your app's state decides which subscriptions, watchers, and sync windows should exist; Trellis computes exactly what to open and what to close.

01

The bug you cannot test for.

Your follow list changes, and somewhere a subscription never closes. The state looks correct. The UI looks correct. The old resource is still alive because a callback missed the one transition where the source set shrank.

A scope closes while a request is in flight. A permission change should drop a market stream. A workspace switch leaves an old sync window open. This class of bug lives in the gap between state and resource ownership, and no unit test reaches it.

Trellis turns each transition into data: a diff, a plan, and a receipt. Watch it happen against a real graph.

02

The receipt is the product surface.

Every committed transaction returns the facts a reviewer needs: changed inputs, recomputed nodes, structural diffs, resource commands, output frames, audit explanations, scope events, and the full phase trace. It is data, not a log line.

03

It checks itself against a full recompute.

Incremental state is presumed guilty. The core crate rebuilds derived state, collections, resource ownership, and outputs from canonical inputs, then asserts that the full recompute equals the incremental graph. The engine ships its own oracle.

let result = tx.commit()?;
for command in result.resource_plan.commands() {
    host.apply(command);
}
graph.assert_incremental_equals_full()?;

04

Adopt it beside your code, not instead of it.

Trellis returns inert data, so it runs next to the reconciliation path you already trust. Feed both the same inputs, compare desired resource state, adjudicate every divergence, and delete the old path last. Shadowing is the adoption method, not a leap of faith.

The first production consumer is a Nostr client framework running exactly this equivalence pattern.

05

A reconciler, not a signal library.

The relatives are the plan phase and the reconcile loop, not observables and query caches. Trellis decides what should happen and hands you the evidence; your host performs the effects.

Terraform planDesired changes as reviewable data before anything executes.
Kubernetes reconcileOwnership, teardown, and convergence, in-process and deterministic.
React commitA commit phase generalized beyond the DOM to external resources.
Elm subscriptionsManaged subscriptions extended to every resource type your host owns.
06

Per-transaction cost is O(total graph state).

That is the deliberate price of structural rollback, deterministic traces, replayable transactions, and a full-recompute oracle. Trellis is a control plane, not a data plane. Keep bulk payloads outside the graph, and the cost model is stated in the open with its own tracked issues.

07

Receipts beat logs for your agents.

Still in development, and the direction is deliberately narrow: hand software agents transaction receipts, trace files, and replayable demos instead of asking them to infer resource ownership from logs. No claim ships here ahead of its demo.
