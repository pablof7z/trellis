Trellis vs signals

Signals are excellent when reads establish value dependencies and changed values rerender views. Trellis starts where those values imply live resources.

Best fit
UI state and view updates.
Application kernels where state owns resources.

Use UI signals for UI state. Use Trellis when source shrink must close resources and clear stale output.
