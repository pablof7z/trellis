Trellis vs controller runtimes

Controller runtimes reconcile desired and actual state at a system boundary. Trellis brings a smaller reconciliation model inside Rust application kernels.

Main unit

Boundary
Often owns long-running reconciliation machinery.
The host actor owns execution; Trellis computes plans.
Best fit
Cluster or platform resource controllers.
Embedded reconciliation inside local app kernels.

Trellis borrows the discipline of reconciliation without becoming an actor framework or scheduler.
