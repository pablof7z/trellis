Guide

Inputs derive collections. Collections produce diffs. Diffs produce plans.

The core API names resource reconciliation as first-class data instead of hiding it inside effect closures.

InputNode<T>

Canonical state changed by the host actor in a transaction.

DerivedNode<T>

A deterministic value computed from declared dependencies.

CollectionNode<K, V>

A derived set or map that emits structural diffs.

ResourcePlan<C>

Plain commands for opening, closing, replacing, clearing, or rebaselining resources.

Scope

The lifetime owner for resources, outputs, and dependency attachments.

Transaction

One propagation cycle with deterministic phase ordering.

MaterializedOutput<O>

Revisioned output frames: baseline, delta, clear, and rebaseline.

FullRecomputeOracle

A testing hook that checks incremental state against truth.

AuditEntry

Diagnostic data explaining changes, plans, ownership, and revisions.

canonical input changed
 -> explicit derived state changed
 -> structural diff produced
 -> resource plan emitted
 -> materialized output revised
 -> full-recompute check remains possible
