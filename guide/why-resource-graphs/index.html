Why signals are not enough.

The hard part in many kernels is not recomputing a value. It is withdrawing obsolete resource demand, installing new demand, clearing stale output, and proving the transition stayed coherent.

The hidden graph

Local-first apps, language servers, dashboards, sync engines, plugin hosts, and market terminals often contain the same shape under different names.

canonical input->derived set->external demand->materialized output

When a source set shrinks, callbacks and handles are not enough. The kernel must know which live resources were implied by the old set and which output rows are no longer valid.

The missing middle

Ordinary value reactivity

A changed, so B recomputes.

Resource graph reactivity

A changed, B derived a new collection, the collection diff produced a resource plan, and a coherent output revision was emitted.

Trellis targets the layer underneath systems where changing state must safely change the resources an application owns.
