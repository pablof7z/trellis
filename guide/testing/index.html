Guide

Correctness is the product.

Trellis should make the bugs that hurt application kernels explicit: source shrink, empty sources, scope teardown, transaction ordering, output revision drift, and incremental/full-recompute mismatch.

incremental == full recompute

Property tests compare current graph state with a fresh recomputation from canonical inputs.

source shrink withdraws

Removed set members produce close commands and stale output clears.

empty fails closed

Empty derived sources produce empty demand, never wildcard demand.

scope close releases

Closing a scope removes ownership and emits teardown plans.

revision is coherent

Output frames carry revisions so consumers can reject stale transitions.

plan idempotence

Repeating unchanged inputs does not reopen already-satisfied resource demand.

given previous = {A, B, C}
and next = {B, C, D}
expect removed = {A}
expect added = {D}
expect plan = [close A, open D]
