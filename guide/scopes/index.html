Guide

Scopes make teardown a graph property.

A resource with no owning scope is a bug. Closing a scope deterministically withdraws ownership and produces close, clear, or rebaseline commands.

scope: dashboard-panel-17
+------------------------------------------------+
| selected_site -> device_set -> topic_plan      |
|                              -> output #42     |
+------------------------------------------------+

Owned resources

Subscriptions, file watchers, jobs, live queries, workers, and output surfaces are attached to scopes.

Shared resources

A resource can stay open while another scope still owns demand for the same key.

Close semantics

Scope close removes ownership, emits close commands for orphaned resources, and clears or rebaselines outputs.

Auditability

Ownership edges are inspectable, so teardown bugs are not hidden in forgotten callbacks.
