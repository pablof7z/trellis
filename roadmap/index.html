Roadmap

Prove the reusable kernel before broadening the surface.

Trellis 0.x is unstable. Minor versions may contain breaking API changes. The public model should stabilize around the vocabulary documented here.

0.1 preview

Explicit input nodes, derived nodes, collection diffs, resource plans, scopes, transactions, revisioned outputs, audit traces, and full-recompute testing hooks.

0.1 launch package

Homepage, README, crate docs, one design essay, three runnable examples, release notes, changelog, roadmap, and comparison page.

0.2 feedback pass

Refine primitive names, error model, observability fields, example ergonomics, and package identity after early feedback.

Stability policy

Before 1.0, breaking changes are allowed but must be explicit in CHANGELOG.md with migration notes.

Adapters

Keep the core independent of async runtimes. Add tracing, serde, tokio, wasm, and proptest support as optional features.

Extraction status

The site sells the category now. The crate API should follow once the core invariants are executable and tested.

Expected stable vocabulary

InputNodeDerivedNodeCollectionNodeResourcePlanScopeTransactionMaterializedOutputRevisionFullRecomputeOracle

The goal is not to solve all reactivity. The goal is to prove one reusable kernel.
